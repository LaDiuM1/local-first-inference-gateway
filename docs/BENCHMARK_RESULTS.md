# 자체 추론 서버 벤치마크 — 공식 측정 결과

> - 문서 상태: 공식 측정 결과
> - 측정일: 2026년 7월 17일
> - 평가 대상: 로컬 생성·임베딩 모델과 서비스 비교 모델
> - 결과 해석과 운영 피드백: [BENCHMARK_FEEDBACK.md](BENCHMARK_FEEDBACK.md)

## 1. 검증 목적

이 벤치마크는 보유한 GPU에 구축한 자체 추론 서버를 실제 서비스의 기본 경로로 사용할 수 있는지
확인하기 위해 수행하였다. 모델 순위가 아니라 다음 질문에 답하는 것이 목적이다.

1. 로컬 생성 모델이 서비스에 필요한 품질과 반응 속도를 제공하는가.
2. 여러 요청이 들어와도 오류나 GPU 메모리 초과 없이 동작하는가.
3. 복잡한 계산과 추론은 로컬 사고 모드로 보완할 수 있는가.
4. 검색 모듈의 임베딩까지 로컬로 전환해도 검색 품질을 유지하는가.
5. 외부 API와 비교했을 때 로컬 운영의 비용 근거가 있는가.

## 2. 측정 조건

### 실행 환경

| 항목 | 조건 |
|---|---|
| GPU | NVIDIA GTX 1080 Ti 11GB |
| 로컬 생성 모델 | `gemma4:12b-it-qat` |
| 로컬 임베딩 모델 | `snowflake-arctic-embed2`, CPU 실행 |
| 비교 생성 모델 | `gpt-5-mini`, `gpt-5.4-nano` |
| 비교 임베딩 모델 | `text-embedding-3-small` |
| 로컬 호출 경로 | 저장소의 FastAPI 게이트웨이 경유 |
| 외부 호출 경로 | OpenAI API 직접 호출 |
| 비교 모델 사고 수준 | 저지연 일반 모드 — `gpt-5-mini`는 `reasoning_effort: minimal`, `gpt-5.4-nano`는 minimal 미지원으로 `none`(필드 생략 시 기본 동작이 none과 동일함을 실측 확인) |
| 로컬 응답 검증 | OpenAI 폴백 비활성화, 성공 응답 전건 `provider=local` 확인 |
| 생성 품질 측정 구성 | 로컬 일반·사고와 두 비교 모델 일반 — `bench_stage9_quality_deep.py --systems local-none,local-high,mini-minimal,nano-none` |
| 측정 시점 | 운영 트래픽이 없는 시간대 — 성능·부하·비용은 심야, 생성 품질은 오전 측정 |

### 평가 데이터

| 평가 | 데이터 구성 | 채점 방식 |
|---|---|---|
| 서비스 생성 품질 — KMMLU | 6개 분야 각 8문항, 총 48문항 | 정답 자동 채점 |
| 서비스 생성 품질 — 자체 고난도 | 다단계 계산 6·논리 3·근거 함정 6·형식 준수 3, 총 18문항 | 정답·형식 자동 채점 + 블라인드 심판 |
| 임베딩 검색 품질 | 커머스 문서 40개·질의 24개 | Recall@1·Recall@3·MRR |
| 단건 생성 성능 | 서비스 시나리오 질의 37건(텍스트 32·이미지 5) | 스트리밍 지연·생성 속도 측정 |
| 동시 부하 | 동시 1·2·4·8건, 출력 상한 120토큰 | 지연·오류·처리량·전력·VRAM 측정 |

서비스 시나리오와 자체 고난도 문항은 프로젝트에서 직접 작성하였다. KMMLU는
[`HAERAE-HUB/KMMLU`](https://huggingface.co/datasets/HAERAE-HUB/KMMLU)의 test 스플릿에서
Computer-Science, Management, Economics, Law, Math, Marketing 분야의 고정 구간을 사용하였다.
KMMLU 원문은 라이선스 조건에 따라 저장소에 재배포하지 않는다.

### 사전 합격 기준

| 영역 | 합격 기준 |
|---|---|
| 안정성 | 동시 4건 이하 부하에서 오류와 GPU 스필오버 0 |
| 체감 성능 | 일반 모드 단건 스트리밍 첫 토큰 p95 5초 이하 |
| 생성 품질 | 블라인드 심판 점수(근거 함정·형식 준수 과제)가 두 비교 모델 각각의 90% 이상 |
| 비용 | 질의당 로컬 한계 전기비가 두 비교 모델의 동일 조건 API 환산액보다 낮음 |

## 3. 공식 측정 결과

### 종합 판정

| 영역 | 측정 결과 | 판정 |
|---|---|:---:|
| 안정성 | 동시 1·2·4·8건 전 구간 오류 0, 최대 VRAM 9,549MiB, GPU 스필오버 없음 | 통과 |
| 체감 성능 | 단건 첫 토큰 p95 2.74초 | 통과 |
| 생성 품질 | 일반 모드 심판 점수 로컬 8.89, `gpt-5-mini` 7.67의 115.9%, `gpt-5.4-nano` 7.11의 125.0% | 통과 |
| 비용 | 로컬 0.028원, `gpt-5-mini` 0.236원, `gpt-5.4-nano` 0.141원/건 | 통과 |

사전에 정한 네 기준을 모두 충족하였다. 이는 로컬 추론 서버를 정상 요청의 기본 경로로 사용하고,
외부 API를 장애와 처리 한계를 보완하는 경로로 유지할 수 있다는 근거다.

### 생성 성능

단건 스트리밍은 32건을 측정했으며 누락은 없었다.

| 지표 | 결과 |
|---|---:|
| 첫 토큰 p50 | 2.55초 |
| 첫 토큰 p95 | 2.74초 |
| 전체 응답 p50 | 3.50초 |
| 생성 속도(첫 토큰 이후 구간 합산) | 30.8 tok/s |
| 이미지 응답 p50 | 2.73초 |
| 사고 모드 첫 토큰 참고값 | 15.03초 |

동시 부하 결과는 다음과 같다.

| 동시 요청 | 완료 요청 | 오류 | 첫 토큰 p50/p95 | 전체 응답 p95 | 총처리량 | 평균 전력 | 최대 VRAM |
|---:|---:|---:|---:|---:|---:|---:|---:|
| 1 | 12 | 0 | 2.45/2.72초 | 6.55초 | 16.6 tok/s | 171.4W | 9,549MiB |
| 2 | 15 | 0 | 6.42/8.61초 | 10.97초 | 15.7 tok/s | 168.2W | 9,549MiB |
| 4 | 15 | 0 | 17.00/20.83초 | 24.70초 | 17.5 tok/s | 185.9W | 9,549MiB |
| 8 | 20 | 0 | 35.61/40.38초 | 42.78초 | 16.6 tok/s | 178.0W | 9,549MiB |

동시 요청이 늘어도 오류나 메모리 초과는 발생하지 않았지만 총처리량은 약 16~18 tok/s로
유지되었다. 요청 수가 늘어날수록 처리량이 증가하기보다 대기 시간이 길어지는 구조임을 확인하였다.

### 서비스 생성 품질

생성 품질은 로컬 일반·사고 모드와 두 비교 모델의 일반 모드를 대상으로 66문항을 측정하였다.
근거 함정과 형식 준수 과제의 심판 점수는 구성명을 가린 뒤 `gpt-5.6-sol`이 채점한 평균이다.

| 평가 | **`gemma4` 일반** | `gemma4` 사고 | `gpt-5-mini` | `gpt-5.4-nano` |
|---|---:|---:|---:|---:|
| KMMLU | 31/48 (64.6%) | **33/48 (68.8%)** | 32/48 (66.7%) | 32/48 (66.7%) |
| 고난도 전체 | 16/18 (88.9%) | **17/18 (94.4%)** | 14/18 (77.8%) | 12/18 (66.7%) |
| 다단계 계산 | 4/6 | **6/6** | **6/6** | 4/6 |
| 논리 | **3/3** | **3/3** | 1/3 | 1/3 |
| 근거 함정 | **6/6** | **6/6** | 5/6 | 5/6 |
| 형식 준수 | **3/3** | 2/3 | 2/3 | 2/3 |
| 함정·형식 심판 점수 | **8.89** | 8.44 | 7.67 | 7.11 |
| 66문항 평균 응답 시간 | 10.3초 | 80.0초 | 2.7초 | 1.9초 |

사고 모드는 계산·논리·근거 함정 전 문항을 맞혔고, 형식 준수 1문항은 정답 내용의 JSON을
코드펜스로 감싸 'JSON 형식으로만 출력' 지시를 어겨 오답 처리하였다. GPT 응답 시간에는
인터넷 왕복이 포함되어 있으므로 로컬과 외부 모델의 직접적인 속도 우열을 판정하는 값으로
사용하지 않는다.

### 임베딩 검색 품질

| 모델 | Recall@1 | Recall@3 | MRR |
|---|---:|---:|---:|
| `snowflake-arctic-embed2` 로컬 | **22/24 (0.92)** | **24/24 (1.00)** | **0.96** |
| `text-embedding-3-small` | 21/24 (0.88) | 23/24 (0.96) | 0.91 |

로컬 임베딩은 CPU에서 텍스트당 약 0.12초로 처리되었다. 원본 출력은 1024차원이며 게이트웨이는
기존 OpenAI 호환 계약을 위해 1536차원으로 zero-padding한다. 이 변환은 코사인 유사도를 보존한다.

### 비용

| 경로 | 질의당 비용 | 산정 기준 |
|---|---:|---|
| 로컬 생성 | **0.028원** | 동시 1 부하의 평균 171.4W에서 유휴 80.9W를 뺀 한계 전력 |
| `gpt-5.4-nano` | 0.141원 | 동일 프롬프트·출력 상한의 실제 usage를 API 단가로 환산 |
| `gpt-5-mini` | 0.236원 | 동일 프롬프트·출력 상한의 실제 usage를 API 단가로 환산 |

환율은 1,500원/USD, 전기요금은 210원/kWh를 적용하였다. API 단가는 2026년 7월 17일
[OpenAI 공식 가격표](https://developers.openai.com/api/docs/pricing)를 기준으로 하였다. 로컬 비용은
이미 보유한 장비에서 요청 한 건을 추가로 처리하는 한계비용이며 장비 구입비·유휴 전력·운영 인건비는
포함하지 않는다.

## 4. 측정 신뢰성

- 로컬 응답은 저장소 게이트웨이를 경유하고 외부 폴백을 비활성화한 상태에서 수집함.
- 성공 응답 전건의 관측 기록에서 `provider=local`을 확인함.
- 생성 품질 66문항은 로컬 일반·사고와 두 비교 모델 모두 수집 실패 0건임.
- 단건·동시 부하 전 구간에서 스트리밍 usage 누락이 없음.
- 정답이 있는 문항은 심판보다 자동 채점을 우선함.
- 생성 심판은 구성명을 가린 응답을 채점함 — 심판 점수는 1~10 정수 범위와 문항·구성
  누락을 검증한 뒤에만 집계함.
- 생성 품질 합격선은 품질 평가 결과(report.json)를 집계기의 `--quality-report`로 연결해
  판정함 — 수집 실패나 채점 누락이 있으면 통과가 아니라 판정 불가로 처리함.
- 비용 비교는 추정 토큰이 아니라 각 API 응답의 실제 usage를 사용함.
- `scripts/verify_stage9.py`가 실 GPU와 유료 API 없이 성능·부하·임베딩·비용 집계와
  품질 합격선 연동을 재현함.

## 5. 해석 범위

- 결과는 GTX 1080 Ti 11GB와 현재 모델·게이트웨이 설정에 대한 실측값임.
- 자체 문항과 검색 셋은 이 프로젝트의 사용 시나리오를 검증하기 위한 것이며 일반적인 모델
  순위를 의미하지 않음.
- 생성 품질의 48개 공개 문항과 18개 자체 문항은 강점과 약점을 진단하기 위한 표본임.
- 이미지 요청은 지연만 측정했으므로 품질 판단에는 별도 평가가 필요함.
- 로컬 비용은 한계 전기비이며 총소유비용과 동일하지 않음.
- 서로 다른 임베딩 provider의 벡터 공간은 호환되지 않으므로 전환과 롤백 모두 전체 재색인이 필요함.
